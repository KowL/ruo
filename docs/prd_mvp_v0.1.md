#### 1. 版本目标
*   用户能够录入股票并查看实时盈亏。
*   用户能够看到该股票当天的 AI 总结新闻，验证“AI 投顾”的价值。
*   **开发周期**：预计 2-3 周。

#### 2. 功能列表 (Feature List)

| 优先级 | ID | 模块 | 功能点描述 | 验收标准 |
| :--- | :--- | :--- | :--- | :--- |
| **P0** | **F-01** | **数据层** | 基础行情接入 | 后端能调通 AkShare/Tushare 接口，获取指定股票的 Code、现价、涨跌幅。 |
| **P0** | **F-02** | **自选管理** | 新增/删除自选股 | 用户输入股票代码（如 600519），系统自动补全名称，保存至数据库。 |
| **P0** | **F-03** | **自选管理** | 持仓信息录入 | 支持输入：持仓成本价、持仓股数、**策略标签**（下拉选：打板/低吸/趋势）。 |
| **P0** | **F-04** | **自选管理** | 首页卡片展示 | 首页列表展示：股票名、现价、**持仓盈亏比**（根据成本价自动计算）。 |
| **P0** | **F-05** | **新闻引擎** | 资讯定时抓取 | 针对自选股列表，每小时爬取/API获取一次最新新闻标题和摘要。 |
| **P0** | **F-06** | **AI 分析** | LLM 情感打分 | 将新闻喂给 LLM，输出：1. 一句话摘要；2. 情感倾向（利好/中性/利空）。 |
| **P1** | **F-07** | **K线分析** | 基础 K 线图 | 点击股票进入详情页，展示日 K 线图（暂不含复杂指标，仅展示 OHLC）。 |
| **P2** | **F-08** | **短线助手** | 竞价数据展示 | (MVP暂不做分析) 仅展示该股当日的开盘金额。 |

#### 3. 核心业务流程 (User Flow)

**场景 A：用户添加一只持仓股**
1.  用户点击“+”号 -> 输入 `000001` -> 系统返回“平安银行”。
2.  用户填写：成本 `10.5`，股数 `1000`，策略标签选 `趋势跟踪`。
3.  点击保存。
4.  首页列表出现卡片，显示当前盈亏状态。

**场景 B：查看 AI 资讯**
1.  用户在首页看到某只股票卡片出现“红色小点”（有新利好）。
2.  点击卡片进入详情页 -> 切换到“情报”Tab。
3.  看到 AI 生成的卡片：
    > **[利好 ★★★★]**
    > **AI 摘要**：平安银行今日发布财报，净利润同比增长 20%，超出市场预期，且不良率进一步降低。
    > *来源：财联社 10:30*

#### 4. 数据库设计简述 (Schema Draft)

*   **Table: `users`** (用户表)
*   **Table: `stocks`** (基础标的表: code, name, market)
*   **Table: `portfolios`** (用户持仓表: user_id, stock_code, cost_price, quantity, strategy_tag)
*   **Table: `stock_news`** (新闻表: stock_code, raw_content, source, publish_time)
*   **Table: `news_analysis`** (AI 分析结果: news_id, ai_summary, sentiment_score, sentiment_label)

#### 5. 待确认问题 (Q&A)
*   **Q**: 实时数据刷新频率？
    *   **A**: MVP 阶段前端手动下拉刷新，或每 30 秒轮询一次，节省服务器资源。
*   **Q**: LLM 使用哪家？
    *   **A**: 开发调试建议用 DeepSeek 或 Kimi 的 API，成本极低且中文财经理解能力强。
